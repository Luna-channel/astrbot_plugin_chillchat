# astrbot_plugin_chillchat

**聊天记录控制** - 上下文字符数裁剪插件

## 功能介绍

这是一个用于控制 LLM 上下文长度的插件。当历史聊天记录的字符数超过设定阈值时，会自动从最旧的消息开始裁剪，避免上下文过长导致的问题：

- 🔧 **节省 Token**：减少不必要的历史记录，降低 API 调用成本
- ⚡ **提升响应速度**：更短的上下文意味着更快的处理速度
- 🛡️ **防止超限**：避免因上下文过长导致的 API 报错
- 🎯 **精准控制**：支持按私聊/群聊分别设置，支持白名单豁免

## 工作原理

1. 在每次 LLM 请求前，计算当前历史记录的总字符数
2. 如果超过设定的 `max_chars` 阈值，从最旧的消息开始逐条删除
3. 保证至少保留最新的一条消息
4. 白名单用户/群聊不受影响

## 配置项

在 AstrBot 管理面板中可配置：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `max_chars` | 历史记录最大字符数 | 8000 |
| `apply_to` | 生效范围：`private`/`group`/`all` | private |
| `whitelist_users` | 白名单用户ID列表 | [] |
| `whitelist_groups` | 白名单群聊ID列表 | [] |

### 配置说明

- **max_chars**：当历史记录总字符数超过此值时触发裁剪。建议根据你使用的模型上下文窗口大小来设置。
- **apply_to**：
  - `private` - 仅对私聊生效
  - `group` - 仅对群聊生效
  - `all` - 对所有会话生效
- **whitelist_users**：这些用户的历史记录不会被裁剪，适合需要长上下文的特定用户
- **whitelist_groups**：这些群聊的历史记录不会被裁剪

## 使用场景

- 私聊用户喜欢长时间连续聊天，历史记录累积过多
- 想要节省 API Token 开销
- 使用上下文窗口较小的模型
- 需要对不同场景（私聊/群聊）采用不同的策略

## 安装

将本插件文件夹放入 AstrBot 的 `addons/plugins/` 目录下，重启 AstrBot 即可。

## 注意事项

- 裁剪操作只影响发送给 LLM 的上下文，不会删除实际的聊天记录
- 至少会保留最新的一条消息，不会完全清空上下文
- 裁剪时会在日志中记录裁剪信息，方便调试
